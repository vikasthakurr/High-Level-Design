# High-Level Design (HLD) Mastery 🚀

Welcome to the **High-Level Design (HLD)** repository. This project is a comprehensive guide and study plan designed to take you from the foundations of system design to advanced distributed patterns used by senior engineers at top-tier tech companies.

---

## 📌 Overview

System Design is not just about choosing a database or a load balancer; it's about understanding trade-offs, anticipating failure, and building systems that can scale gracefully. This repository contains a structured roadmap, case studies, and deep dives into the core components of modern architecture.

## 🗺️ Roadmap

The curriculum is divided into four main phases:

### 🏗️ Phase 1: Foundations of System Design
- **Module 1**: What Is HLD? (Design Mindset, Functional vs Non-functional requirements)
- **Module 2**: Networking & Communication (HTTP/3, gRPC, WebSockets, Load Balancers)
- **Module 3**: Storage Systems (SQL vs NoSQL, Indexing, Sharding, CAP Theorem)
- **Module 4**: Caching (Strategies, Invalidation, Redis Architecture)

### 🌐 Phase 2: Core Distributed Systems
- **Module 5**: Scalability Patterns (Rate Limiting, Backpressure, Bulkheads)
- **Module 6**: Event-Driven Architecture (Message Queues, Kafka, Saga Pattern, CQRS)
- **Module 7**: Distributed Systems Fundamentals (Consensus, Paxos/Raft, Consistency Models)
- **Module 8**: Search Systems (Elasticsearch, Vector Search, Typeahead)

### 🧩 Phase 3: Advanced Distributed Patterns
- **Module 9**: Microservices Architecture (DDD, Service Mesh, Observability)
- **Module 10**: Micro Frontend Architecture (Module Federation, Shared Design Systems)
- **Module 11**: Data-Intensive System Design (Stream Processing, Data Pipelines)
- **Module 12**: Notification & Communication Systems

### 🛡️ Phase 4: Mastery & Reliability
- **Module 13**: Reliability Engineering (Chaos Engineering, Failover, SLA/SLO)
- **Module 14**: Security in System Design (OAuth 2.0, JWT, Common Attack Vectors)
- **Module 15**: Cost-Aware System Design
- **Module 16**: Capstone Projects (Designing YouTube, Uber, Figma, and more)

## 🔬 Case Studies

Learn from real-world architecture evolutions:
- **Instagram**: From a single Postgres instance to a globally distributed system.
- **Netflix**: Global video delivery and CDN architecture.
- **Uber**: Surge pricing engine and geospatial aggregation.
- **WhatsApp**: Scaling messaging to 100B+ messages/day.
- **Google Spanner**: Achieving global strong consistency.
